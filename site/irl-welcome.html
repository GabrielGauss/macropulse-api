Quickstart

Deploy in 3 steps

01

Install the IRL SDK

Add the MacroPulse IRL client library to your agent environment.

pip install macropulse-irl

# or with npm for Node/TypeScript agents
npm install @macropulse/irl-sdk
02

Set your license key

The key is in your welcome email. Set it as an environment variable — never hardcode it in source.

# .env or container env
MACROPULSE_IRL_KEY=mp_your_license_key_here
MACROPULSE_MTA_URL=https://api.macropulse.live
03

Wrap your agent's reasoning

Before any order placement, call irl.seal() with your agent's reasoning. The IRL Engine fetches the current MTA regime, seals the record, and returns a receipt_id for audit linkage.

# Python — minimal integration
from macropulse_irl import IRLClient

irl = IRLClient()  # reads MACROPULSE_IRL_KEY from env

# Before placing any order:
receipt = irl.seal(
    agent_id="agent-fx-001",
    reasoning=agent.last_chain_of_thought(),
    proposed_order=order_dict,
)

# Attach receipt_id to your order record
order.irl_receipt_id = receipt.receipt_id

// TypeScript — equivalent
import { IRLClient } from '@macropulse/irl-sdk';

const irl = new IRLClient();  // reads MACROPULSE_IRL_KEY
const receipt = await irl.seal({
  agentId: 'agent-fx-001',
  reasoning: agent.lastChainOfThought(),
  proposedOrder: orderPayload,
});
order.irlReceiptId = receipt.receiptId;
L2 Audit Platform

Minimum 3-agent requirement

Your IRL Audit Platform license covers 3 autonomous agents. Each agent instance must be identifiable by a unique agent_id in every irl.seal() call.

To add more agents beyond your current count, update your subscription quantity via the billing portal — your agent count on the license is adjusted automatically at the next billing cycle.

# Multi-agent deployment — each agent uses the same key, unique agent_id
agents = [
    IRLClient(agent_id="fx-agent-01"),
    IRLClient(agent_id="eq-agent-01"),
    IRLClient(agent_id="macro-agent-01"),
]

Need a 4th agent? Simply increase your subscription quantity in the billing portal. Billing adjusts prorated from the change date.

License Terms

Product regulations

License Restrictions Non-negotiable

  • Non-transferable. This license key is issued to the purchasing entity and may not be shared, sold, sublicensed, or transferred to any third party. Detected transfers result in immediate revocation.
  • Single deployment context. The license is scoped to your registered organisation. Cross-organisation deployment (e.g. managing clients' agents under one key) requires a separate license per client.
  • Agent count enforcement. You may run up to the licensed number of simultaneous agents. Running more agents than licensed is a contract violation and may trigger automated revocation.
  • No credential exposure. Do not log, commit, or transmit the license key. Store it in a secrets manager (AWS Secrets Manager, Vault, GCP Secret Manager, etc.), not in environment files committed to version control.
  • Payment continuity. Access is suspended automatically if a payment fails. Stripe retries three times over 7 days. If all retries fail, the key enters suspended state — reactivated automatically once a valid payment is processed.
  • Audit trail integrity. IRL receipt records are immutable. Tampering with, suppressing, or selectively submitting agent reasoning before order placement defeats the purpose of the product and constitutes a terms violation.
  • No reverse engineering. You may not reverse engineer, decompile, or extract the MacroPulse MTA model artifacts, PCA components, or HMM parameters from API responses.
