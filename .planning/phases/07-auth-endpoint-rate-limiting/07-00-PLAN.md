    Create api/middleware/auth_rate_limit.py. This is a new file — do not modify rate_limit.py.

    The module contains three public functions:
    1. get_client_ip(request: Request) -> str
    2. check_auth_rate_limit(identifier, endpoint, max_attempts, window_minutes, with_backoff=False) -> None
    3. _set_backoff_if_needed(identifier, endpoint, attempt_count) -> None (private, used by check_auth_rate_limit)

    _BACKOFF_SCHEDULE = {3: 30, 4: 60, 5: 300}  # attempt_count → lockout seconds
    For attempt_count > 5, backoff is 300 seconds.

    get_client_ip() replicates the exact logic from rate_limit.py lines 130–139 (trusts X-Forwarded-For only when direct_host.startswith("172.18.")).

    check_auth_rate_limit():
    - Calls queries.check_and_record_attempt() inside try/except Exception
    - On exception: logger.error(...) then return (fail open — do NOT raise)
    - If with_backoff and result["allowed"] and result["attempt_count"] >= 3: call _set_backoff_if_needed()
    - If not result["allowed"]: compute retry_after from locked_until (or default 60), raise HTTPException(429)

    _set_backoff_if_needed():
    - Looks up backoff seconds from _BACKOFF_SCHEDULE (or 300 if count > 5, 0 if count < 3)
    - If backoff == 0: return immediately
    - Runs UPDATE auth_rate_limits SET locked_until = now() + %s::interval WHERE identifier = %s AND endpoint = %s
    - Wrap in try/except: log error, do not re-raise
