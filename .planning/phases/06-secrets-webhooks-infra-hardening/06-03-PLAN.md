<objective>
Lock down the three infrastructure attack surfaces: read-only model artifact mount, HTTP security header (CSP), and CORS wildcard startup guard.

Purpose: The API container currently has write access to the model_artifacts volume — an attacker with RCE can replace model files and inject false trading signals. The nginx config lacks a Content-Security-Policy header. And CORS_ORIGINS can be set to wildcard '*' in production without any startup-time rejection. This plan closes all three gaps.
Output: :ro volume flag in docker-compose.yml, CSP add_header in nginx.conf, _validate_cors_origins() in lifespan(), test_security.py converted from xfail to real assertion.
</objective>

**nginx/nginx.conf — add Content-Security-Policy header (SEC-41):**

In the HTTPS server block, add the CSP header immediately after the existing three security headers (after X-Frame-Options line):
```nginx
    add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; connect-src 'self' wss://api.macropulse.live https://api.macropulse.live; font-src 'self'; frame-ancestors 'none'; base-uri 'self'" always;
```

Notes on the CSP directives:
- `default-src 'self'` — baseline restriction to same origin
- `script-src 'self'` — Vite builds hashed JS bundles, no inline scripts needed
- `style-src 'self' 'unsafe-inline'` — React component libraries may inject inline styles
- `img-src 'self' data: https:` — allows data URIs and HTTPS images (charts, icons)
- `connect-src 'self' wss://api.macropulse.live https://api.macropulse.live` — WebSocket stream + REST API
- `font-src 'self'` — local fonts only
- `frame-ancestors 'none'` — prevents clickjacking; replaces X-Frame-Options SAMEORIGIN (keep both for compat)
- `base-uri 'self'` — prevents base tag injection

FUTURE NOTE (add as comment in nginx.conf near the CSP line): When Phase 10 (Paddle Billing) is implemented, add `frame-src https://checkout.paddle.com` to the CSP to allow the Paddle checkout iframe.

def _validate_cors_origins() -> None:
    """Raise at startup if CORS wildcard is configured in production."""
    settings = get_settings()
    if settings.env == "production" and "*" in settings.cors_origins:
        raise RuntimeError(
            "CORS wildcard '*' is not allowed in production. "
            "Set CORS_ORIGINS to explicit allowed origins (e.g., https://macropulse.live) "
            "in .env or via environment injection before starting."
        )
```

"""
Startup security guard tests — Phase 6 (SEC-42).
"""
import os
import pytest
from unittest.mock import patch


def test_cors_wildcard_blocked_in_prod():
    """App raises RuntimeError at startup if ENV=production and CORS_ORIGINS contains '*'."""
    from config.settings import get_settings
    from api.main import _validate_cors_origins

    # Production + wildcard → must raise
    get_settings.cache_clear()
    with patch.dict(os.environ, {"ENV": "production", "CORS_ORIGINS": '["*"]'}, clear=False):
        get_settings.cache_clear()
        with pytest.raises(RuntimeError, match="CORS wildcard"):
            _validate_cors_origins()

    # Development + wildcard → must NOT raise (dev is allowed to be permissive)
    get_settings.cache_clear()
    with patch.dict(os.environ, {"ENV": "development", "CORS_ORIGINS": '["*"]'}, clear=False):
        get_settings.cache_clear()
        _validate_cors_origins()  # should not raise

    get_settings.cache_clear()
```

Note on CORS_ORIGINS env var format: pydantic-settings parses list[str] from a JSON-encoded string when set as an env var (e.g., CORS_ORIGINS='["*"]'). Verify this is correct for the installed pydantic-settings version by checking how cors_origins is currently used. If the format differs, adjust the test's patch.dict value accordingly.
