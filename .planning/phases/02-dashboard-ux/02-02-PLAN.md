- Inside the effect that calls `setViewDays(isFree ? 30 : 365)`, add `if (tier === null) return;` as the first line

**Change 2 — App.jsx:**
At the RegimeCalendar call site (search for `<RegimeCalendar`), add the `tier` prop:
- Change `<RegimeCalendar isFree={isFree}` to `<RegimeCalendar isFree={isFree} tier={tier}`

The `tier` state variable already exists in App.jsx (`useState(null)`). No new state or imports needed.

**Why this works:** On mount, `tier=null` → the effect returns early, leaving `viewDays=365` (the correct default). When `api.getMe()` resolves and sets `tier='owner'`, the effect re-fires with `tier !== null` and `isFree=false` → `setViewDays(365)` (no visible change). For free users: `tier='free'` → `isFree=true` → `setViewDays(30)` (correct).
